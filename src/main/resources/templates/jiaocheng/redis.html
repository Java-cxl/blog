redis的安装和使用

redis安装启动

1.安装redis需要c语言的编译环境。如果没有gcc需要在线安装。

 
 yum install gcc-c++

2.把redis源码包上传到linux

3.解压缩redis源码包

 
xxxxxxxxxx
tar -zxf redis.x.x

4.进入解压后的redis文件夹中，然后使用“make”命令编译

5.安装redis

 
xxxxxxxxxx
[root@weitao redis-3.0.0]# make install PREFIX=/usr/local/redis
//PREFIX指定redis要安装的目录

6.启动redis

前台启动：在redis的安装目录下直接启动redis-server（一直处于启动页面，如果退出将会终止redis，不推荐）

 
xxxxxxxxxx
[root@localhost bin]# ./redis-server

后台启动：（推荐）

把redis-3.0.0/redis.conf（解压后的redis目录下）复制到/usr/local/redis/bin（redis的安装目录下）目录下

 
xxxxxxxxxx
[root@localhost redis-3.0.0]# cp redis.conf    /usr/local/redis/bin/

然后更改redis.conf文件，daemonize no 改为 daemonize yes

（redis采用的是单进程多线程的模式。当redis.conf中选项daemonize设置成yes时，代表开启守护进程模式。在该模式下，redis会在后台运行，并将进程pid号写入至redis.conf选项pidfile设置的文件中，此时redis将一直运行，除非手动kill该进程。但当daemonize选项设置成no时，当前界面将进入redis的命令行界面，exit强制退出或者关闭连接工具(putty,xshell等)都会导致redis进程退出。 服务端开发的大部分应用都是采用后台运行的模式）

最后就可以启动redis了

 
xxxxxxxxxx
[root@localhost bin]# ./redis-server redis.conf

7.查看redis进程

 
xxxxxxxxxx
[root@localhost bin]# ps aux|grep redis

8.redis客户端连接服务端

 
xxxxxxxxxx
//默认使用127.0.0.1 6379端口
[root@localhost bin]# ./redis-cli
//-h：连接的服务器的地址   -p：服务的端口号
[root@localhost bin]# ./redis-cli -h 192.168.25.153 -p 6379
Redis的五种基本数据类型

redis是以 key / value 形式存储数据的，数据类型指的是value的数据类型

redis中所有数据都是字符串类型

1.string 

 
xxxxxxxxxx
添加数据：set key1 value1 ；         获取数据：get key1
把key1对应的值加1: incr key1 ；      把key1对应的值减1：decr key1

2.hash 

 
xxxxxxxxxx
添加数据：hset hash1 field1 1               获取数据：hget hash1 field1
获取指定数据类型的所有键：hkeys hash1        获取指定数据类型的所有值：hvals hash1
获取指定数据类型的所有键和值：hgetall hash1   删除键：hdel hash1 field1

3.list （可以重复，元素有放入顺序）

 
xxxxxxxxxx
从左开始添加数据：lpush list1 1 2 3 4 5 6  //添加完是6，5，4，3，2，1
从右开始添加数据：rpush list1 a b c d e    //添加完是 a,b,c,d,e
查询list1所有值：lrange list1 0 -1 //起始下标和结束下标 0 -1表示查询全部 6,5,4,3,2,1,a,b,c,d,e
从最左边取一个值：lpop list1 //取完值就没了，取完还剩 5,4,3,2,1,a,b,c,d,e 
从最右边取一个值：rpop list1 //取完值就没了，取完还剩 5,4,3,2,1,a,b,c,d

4.set（没有顺序，不可以重复）

 
xxxxxxxxxx
添加数据：sadd set1 a b a b c d //存入只有a b c d
删除数据：srem set1 a
查看数据：smembers set1 // b c d
sadd seta a b c d e   sadd seta c d e f g
两列数据相减：sdiff seta setb //得到a b
取两列数据的交集：sinter seta setb //得到 c d e
取两列数据的并集：sunion seta  setb //得到 a b c d e f g

5.sorted set

设置key的有效时间 ：expire key1 100

查看key的有效时间：ttl key1

ttl命令返回的数字如果是整数代表正在倒计时，如果是-1，代表该key是永久保存的，如果是-2代表key不存在

Redis搭建集群环境

Redis集群中至少应该有三个节点。要保证集群的高可用，需要每个节点有一个备份机。

所以Redis集群至少需要6台服务器。

搭建伪分布式。可以使用一台虚拟机运行6个redis实例。需要修改redis的端口号7001-7006

1，在usr/local目录下创建目录redis-cluster

2，进入前面安装好的redis目录，复制bin目录到/usr/local/redis-cluster目录下

 
xxxxxxxxxx
[root@weitao redis]# cp -r bin /usr/local/redis-cluster/redis01

3，进入到复制好的redis01的bin目录，删除 dump.rdb 和 appendonly.aof 以保证节点是新的。然后修改redis.conf配置文件，修改port为7001；再去掉 #cluster-enabled yes 的注释(大约632行)，以便搭建集群

4，按照2，3两个步骤，复制共六份，端口分别改为（7001，7002，7003，7004，7005，7006）

5，编写一个批处理，让六个redis依次启动

 
xxxxxxxxxx
[root@weitao redis-cluster]# vi start-all.sh

cd ..

./redis-server redis.conf
cd ..
cd redis04
./redis-server redis.conf
cd ..

./redis-server redis.conf
cd ..

./redis-server redis.conf

此时的 start-all.sh 文件只是一个普通的文件，通过下面的代码将它改为可执行文件

 
xxxxxxxxxx
[root@weitao redis-cluster]# chmod u+x start-all.sh

6，执行批处理文件，启动六个redis

 
xxxxxxxxxx
[root@weitao redis-cluster]# ./start-all.sh

xxxxxxxxxx
[root@weitao src]# cp redis-trib.rb /usr/local/redis-cluster/

2，使用ruby脚本搭建集群。需要在线安装ruby的运行环境：

先把redis-3.0.0.gem传到linux中，然后进行安装：

 
xxxxxxxxxx
[root@weitao local]# gem install redis-3.0.0.gem

xxxxxxxxxx
[root@weitao redis-cluster]# ./redis-trib.rb create --replicas 1 192.168.25.16:7001 192.168.25.16:7002 192.168.25.16:7003 192.168.25.16:7004 192.168.25.16:7005 192.168.25.16:7006
使用集群
 
xxxxxxxxxx
[root@weitao redis-cluster]# redis01/redis-cli -p 7001 -c
//-c：代表连接的是redis集群

查询业务，首先根据key查询redis缓存，如果缓存中有，直接转换为需要的数据类型然后返回。如果缓存中不存在数据，则查询之后放入redis缓存，最后返回。代码示例：

 
xxxxxxxxxx
public List<TbContent> getContentList(long id) {
    //查询redis缓存中是否存在数据
    try {
        String value=jedisClient.hget(CONTENT_LIST,id+"");
        if(StringUtils.isNotEmpty(value)){
            List<TbContent> redisList= JsonUtils.jsonToList(value, TbContent.class);
            System.out.println("查询缓存得到的值");
            return redisList;
        }
    } catch (Exception e) {
        // TODO: handle exception
        e.printStackTrace();
    }
    TbContentExample example=new TbContentExample();
    Criteria criteria=example.createCriteria();
    criteria.andCategoryIdEqualTo(id);
    List<TbContent> list=contentMapper.selectByExampleWithBLOBs(example);
    //查询出来的数据向redis中添加
    try {
        jedisClient.hset(CONTENT_LIST, id+"", JsonUtils.objectToJson(list));
        System.out.println("查出来向redis中添加!");
    } catch (Exception e) {
        // TODO: handle exception
        e.printStackTrace();
    }
    return list;
}

增删改业务需要删除redis中相应的缓存以便实现缓存同步

示例代码：

 
xxxxxxxxxx
public TaotaoResult addContent(TbContent content) {
    content.setCreated(new Date());
    content.setUpdated(new Date());
    contentMapper.insert(content);
    //根据key删除redis中缓存以实现缓存同步
    jedisClient.hdel(CONTENT_LIST, content.getCategoryId().toString());
    return TaotaoResult.ok();
}
